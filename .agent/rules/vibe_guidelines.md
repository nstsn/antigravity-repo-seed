# Vibe / UI & Copy Guidelines

## Core Vibe
- 「ミニマルで信頼できる業務ツール」
- 余計な装飾はしないが、状態（成功/失敗/危険）は分かりやすく出す
- クリック数を減らすため、候補は “チップ” で提示

## UIトーン
- ラベルは短く、補足は小さく（例：`プレビュー` / `元に戻す` / `タグ候補`）
- エラー文は原因と次の行動を1行ずつ：
  - 例：「リネームUIが見つかりませんでした（サイト側UI変更の可能性）。ページを再読み込みして再試行してください。」

## レイアウト方針（Side Panel）
上から：
1) 現在のスレッド状態（サイト/取得できたタイトル/URL）
2) リネーム操作（本文、タグ、プレビュー、実行、Undo）
3) リネーム済み検索（タグ/期間/検索語 → 結果 → ジャンプ）

## インタラクション
- タグ候補チップ：クリックで追加、再クリックで解除
- “タグなし” を常に選べる
- 成功トースト＋Undoボタンは一定時間表示（例：10秒）
- ローディングは「実行中…」だけで良い（過剰アニメ不要）

## アクセシビリティ
- キーボード操作を優先：
  - 入力欄フォーカス、Enterで実行、Escで閉じる（可能なら）
- 色だけに依存しない（成功/警告/失敗のアイコンや文言を併用）
